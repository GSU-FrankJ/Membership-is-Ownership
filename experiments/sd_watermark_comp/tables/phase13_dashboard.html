All criteria use raw t-error scores on watermark set W. Ownership verified only when all three pass.

IV. Key Findings
Finding 1

Algorithm 2 does not work for any SD fine-tuning

Raw score ratio is ~1.01–1.03x across all configurations (LoRA r64, r256, full FT). The base model's per-image reconstruction variance (~0.022) is 7x the membership signal (~0.003). Algorithm 2's thresholds were calibrated for from-scratch DDIM where ratio exceeds 19x.

A7 full FT: ratio=1.028x, |d|=0.14 (need >5.0x and >2.0)
Finding 2

LoRA r256 achieves the strongest membership detection

Higher rank increases separation magnitude. A8 (r256, 15.6% params) outperforms both A6 (r64, 3.9%) and A7 (full FT, 100%) on Cohen's d.

A6 d=3.26 → A8 d=3.93 → A7 d=3.41 | A8 AUC=0.9994, TPR@1%=0.985
Finding 3

C1 correctly distinguishes model lineage

B1/B2 were derived from A6 (LoRA r64), not from A7 or A8. C1 correctly reflects this: only A6 passes consistency with B1/B2. A7 and A8 fail because they are different models with different raw t-error characteristics, not because adversary modification is detected.

A6 vs B2: p=0.337 PASS (correct — same lineage) | A7 vs B2: p=0.021 FAIL (correct — different lineage) | A8 vs B2: p=0.042 FAIL (correct — different lineage)
Finding 4

Delta-based protocol is needed for SD models

The membership signal lives in delta = score_tgt − score_ref, not in raw scores. Delta Cohen's d = 3.93 (strong) vs raw d = 0.13 (invisible). An adapted verification protocol with recalibrated thresholds is required.

Delta signal: d=3.93, AUC=0.9994 | Raw signal: d=0.13, ratio=1.025x
